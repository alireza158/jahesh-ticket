# سامانه ثبت و مدیریت تیکت پشتیبانی جهش

این پروژه یک پنل پشتیبانی Laravel + Bootstrap RTL برای شرکت جهش است و نقش‌های زیر را پوشش می‌دهد:

- **مدیر مجموعه:** مدیریت مشتریان، پروژه‌ها، اکانت مشتری، پرسنل، پرداخت‌ها و گزارش مشتریان بدهکار.
- **مدیر وبسایت:** مشاهده همه تیکت‌ها، ارجاع تیکت به پرسنل همراه یادداشت، تغییر وضعیت و بستن تیکت.
- **پرسنل:** مشاهده فقط تیکت‌های ارجاع‌شده به خودش، پاسخ به تیکت، ارسال فایل، تغییر وضعیت و ثبت یادداشت داخلی.
- **مشتری:** ورود به پنل، ثبت تیکت با عنوان/توضیحات/فایل/اولویت/شماره تماس، مشاهده تیکت و وضعیت، ثبت پرداخت ماهانه و مشاهده وضعیت پرداخت.

## نیازمندی‌ها

- PHP 8.2+
- Composer
- Node.js و npm
- SQLite یا هر دیتابیس پشتیبانی‌شده توسط Laravel

## راه‌اندازی از صفر

```bash
composer install
cp .env.example .env
php artisan key:generate
touch database/database.sqlite
php artisan migrate --seed
php artisan storage:link
php artisan optimize:clear
npm install
npm run build
php artisan serve
```

بعد از اجرای Seed، داده‌های دمو شامل مدیر، مدیر وبسایت، پرسنل، مشتریان، پروژه‌ها، تیکت‌ها و پرداخت‌ها ساخته می‌شود. ورود فقط با شماره تماس انجام می‌شود:

- شماره تماس مدیر: `09111111111`
- شماره تماس مدیر وبسایت دمو: `09112223344`
- شماره تماس پرسنل دمو: `09120000001`، `09120000002`، `09120000003`
- شماره تماس مشتریان دمو: `09130000001`، `09130000002`، `09130000003`
- رمز عبور همه اکانت‌های Seed: `12345678`

## اجرای محیط توسعه

```bash
composer run dev
```

یا به صورت جداگانه:

```bash
php artisan optimize:clear
php artisan serve
npm run dev
```

## اجرای تست‌ها و بررسی کد

```bash
php artisan test
./vendor/bin/pint --test
```

## نکات پیاده‌سازی

- فایل‌ها و رسیدها روی disk عمومی ذخیره می‌شوند؛ برای دسترسی از وب باید `php artisan storage:link` اجرا شود.
- ارسال پیامک فعلاً در `storage/logs/laravel.log` لاگ می‌شود و محل اتصال API پنل پیامکی در `app/Services/SmsService.php` آماده است.
- تاریخ‌های قابل نمایش و ورودی پرداخت با تقویم شمسی کار می‌کنند؛ تاریخ شمسی پرداخت در سمت سرور به تاریخ میلادی دیتابیس تبدیل می‌شود.
- Bootstrap RTL و datepicker شمسی از CDN بارگذاری می‌شوند.
- مدیر مجموعه و مدیر وبسایت می‌توانند از مسیر پرداخت‌ها برای پروژه‌ها پرداخت ثبت کنند.
- هر پروژه هزینه اولیه، هزینه ماهانه، بدهکاری دستی، بستانکاری دستی، یادداشت مالی و گزارش پرداخت‌های همان پروژه را دارد.
- مشتری در پنل پرداخت‌های خود مانده بدهی و بستانکاری هر پروژه و جمع کل را مشاهده می‌کند.
